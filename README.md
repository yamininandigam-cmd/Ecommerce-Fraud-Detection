# 🛡️ E-commerce Fraud Detection System

## 📌 Overview

This project focuses on detecting fraudulent transactions in e-commerce using machine learning techniques. Due to the highly imbalanced nature of fraud data, special attention was given to improving detection of rare fraudulent cases.

---

## 🎯 Objective

* Detect fraudulent transactions accurately
* Handle class imbalance effectively
* Improve recall for fraud detection

---

## 🧠 Approach

* Data preprocessing and cleaning
* Exploratory Data Analysis (EDA)
* Handling imbalanced dataset using SMOTE
* Training and evaluating classification models

---

## ⚖️ Handling Imbalanced Data

* Applied **SMOTE (Synthetic Minority Oversampling Technique)**
* Balanced fraud and non-fraud classes
* Improved model performance on minority (fraud) class

---

## 🤖 Classification Models Used

* Logistic Regression
* Random Forest 

---

## 📊 Evaluation Metrics

* Accuracy
* Precision
* Recall (important for fraud detection)
* F1-score

> Special focus was given to **recall score** to ensure fraudulent transactions are correctly identified.

---

## 🛠️ Tech Stack

* Python
* Pandas, NumPy
* Scikit-learn
* Imbalanced-learn (SMOTE)
* Matplotlib, Seaborn

---

## 📈 Key Learnings

* Importance of handling imbalanced datasets
* Trade-off between precision and recall
* Real-world challenges in fraud detection systems

---
